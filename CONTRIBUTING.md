# Contributing to the Project

Thank you for your interest in contributing to this project ❤️

Please read this document carefully before making any changes.

## Contribution Process

### Step 1: Fork the Project

Fork this repository to your GitHub account and develop your feature or bug fix in your own fork.

### Step 2: Complete Your Changes

Before submitting your work:

* Review your code.
* Ensure the project still functions correctly.
* Commit all necessary changes.

### Step 3: Notify the Development Team

**DO NOT create a Pull Request immediately.**

Before submitting a Pull Request, you must:

1. Join the project's Support Center: https://discord.gg/BpwQS556Zj
2. Create a new request for Pull Request review.
3. Provide a brief description of your changes.

Once your request has been received, the development team will issue you a **PR ID**.

### Step 4: Create a Pull Request

When creating your Pull Request:

* The title or description must include the assigned PR ID.
* Clearly describe the changes you made.
* List any bugs fixed or features added.

Pull Requests that do not contain a valid PR ID may be rejected or closed without review.

**Receiving a PR ID does not guarantee that your Pull Request will be accepted.**

### Step 5: Review Process

After your Pull Request has been submitted:

* Staff will confirm that your request has been received.
* The development team will review your code.
* You will be notified once a decision has been made regarding acceptance or rejection.

## Contributor Benefits

If your Pull Request is accepted:

* You will receive the **Contributor** role in the Discord Community Server: https://discord.gg/C4ZANNdYfs
* You will be officially recognized as a contributor to the project.

In some cases, Contributors may be invited to join the development team without going through the standard skill evaluation process, as their contributions have already demonstrated their abilities and suitability for the project.

## Notes

* Please read `CODE_OF_CONDUCT.md` before contributing.
* Respect other community members.
* Do not submit malicious code, backdoors, or any harmful content.
* The development team reserves the right to reject any Pull Request that does not align with the project's goals or standards.
